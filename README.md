# Network-based IDS & Security Monitoring

## Overview
This project implements a **network-based intrusion detection and security monitoring system**
in a virtualized network environment.
It focuses on **monitoring network traffic, detecting malicious activities**, and
**centralizing security logs for analysis and visualization**.

The system uses **Suricata** as a Network-based IDS.
Logs generated by Suricata are collected using **Filebeat** and forwarded to
**Elasticsearch**, where they are analyzed and visualized using **Kibana**.

---

## Objectives
- Design and deploy a virtual network environment
- Monitor and analyze network traffic
- Detect suspicious or malicious activities using a network-based IDS
- Centralize and visualize security logs

---

## Environment & Tools
- **Network Emulator**: GNS3
- **Servers**
  - Linux Web Server
  - Linux DNS Server
- **IDS**: Suricata
- **Log Shipper**: Filebeat
- **Log Storage & Visualization**
  - Elasticsearch
  - Kibana

---

## Project Architecture
The virtual network consists of multiple Linux servers and a dedicated IDS sensor.
Network traffic is mirrored to the IDS, where Suricata inspects packets and generates alerts.

Security logs generated by Suricata are shipped via Filebeat to Elasticsearch.
Kibana is used to visualize alerts, traffic patterns, and security events.

Network topology diagrams are available in the **`04-network-diagram/`** directory.

---

## Results
- Successfully detected simulated attack traffic using Suricata
- Collected and centralized IDS logs using Filebeat and Elasticsearch
- Visualized security events and network traffic patterns through Kibana dashboards

The system demonstrated effective network traffic monitoring and intrusion detection
within a virtualized environment.
